# ⚡ Hydrogen Production Optimizer
**Group 4.04 — "Not Found" | Fundamentals of Computer Science · HS 2026**

A Streamlit web app that helps green hydrogen producers minimize electricity costs by predicting price movements in the Australian NEM and scheduling production during the cheapest hours.

---

## 🚀 Getting Started

**1. Install dependencies**
```bash
pip install -r requirements.txt
```

**2. Run the app**
```bash
streamlit run app.py
```

The app will open in your browser automatically. Use the sidebar to navigate between pages.

---

## 📁 Folder Structure

```
hydrogen-optimizer/
│
├── app.py                        # Entry point — landing page with overview metrics
├── requirements.txt              # All Python packages needed to run the app
├── .gitignore                    # Files Git should not track (caches, large data, etc.)
│
├── .streamlit/
│   └── config.toml               # App theme and layout settings
│
├── pages/                        # Each file here = one page in the Streamlit sidebar
│   ├── 1_Market_Overview.py      # Live electricity prices + weather charts + news feed
│   ├── 2_Price_Forecast.py       # ML price predictions with confidence bands
│   ├── 3_Production_Optimizer.py # User inputs + optimal production schedule
│   └── 4_Cost_Analysis.py        # Savings comparison, break-even analysis, CSV export
│
├── utils/                        # Shared logic — imported by the pages above
│   ├── __init__.py               # Makes utils/ a Python package (do not delete)
│   ├── api.py                    # All external API calls (AEMO, weather, news)
│   ├── model.py                  # ML pipeline: feature engineering, training, prediction
│   ├── optimizer.py              # Production scheduling algorithm
│   └── helpers.py                # Constants (regions, defaults) and formatting functions
│
└── data/
    └── sample_prices.csv         # Fake price data for local development and testing
```

---

## 🧭 How the Pages Connect

```
1_Market_Overview      →   pulls data from   →   utils/api.py
2_Price_Forecast       →   pulls data from   →   utils/api.py
                       →   trains model via  →   utils/model.py
                       →   stores forecast in Streamlit session state
3_Production_Optimizer →   reads forecast from session state
                       →   runs schedule via →   utils/optimizer.py
                       →   stores results in session state
4_Cost_Analysis        →   reads results from session state
```

> **Important:** Pages 3 and 4 depend on data generated by Page 2.
> Always run the forecast on Page 2 first before using the Optimizer.

## 📐 Key Convention: Shared Data Formats

To make sure modules work together, we use these standardised column names everywhere.
**Do not rename these columns** — other files depend on them.

| DataFrame | Required Columns |
|-----------|-----------------|
| Electricity prices (raw) | `timestamp`, `price_aud_mwh` |
| Weather data | `timestamp`, `temperature_c`, `wind_speed_kmh`, `solar_radiation_wm2` |
| Price forecast (model output) | `timestamp`, `predicted_price_aud_mwh` |
| Production schedule (optimizer output) | `timestamp`, `predicted_price_aud_mwh`, `produce`, `h2_produced_kg`, `electricity_cost_aud` |
